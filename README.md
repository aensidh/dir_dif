dir_diff.sh — автоматический анализ изменений в проекте после сборки
./dir_dif <путь к проекту> <команда для сборки>

Также возможно использование с флагом -o DIR или --output DIR
./dir_diff.sh <путь_к_проекту> '<команда_сборки>' <-o или --output> <DIR>

Пример с nginx:
	sudo ./dir_diff.sh nginx "./configure && make"

Что делает:
  1. dir_diff создает временную копию директории проекта
  2. Вы полняется сборка проекта в основной директории
  3. Сравниваются файлы(новый файл или старый)
  4. С помощью cmp сравниваются файлы по битово и в случае, если файл изменен, он сохранияется
  5. Очистка временных директорий
